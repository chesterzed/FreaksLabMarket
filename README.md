# FreakLabMarket

FreakLabMarket — это веб-приложение интерактивной галереи, разработанное на базе фреймворка Django. Проект ориентирован на создание удобной площадки для демонстрации продукции с современным интерфейсом и расширенными профилями пользователей.

## Основной функционал

- **Каталог товаров (`goods`):** Просмотр товаров, фильтрация по категориям, детальные страницы товаров.
- **Пользователи (`users`):** Регистрация, авторизация, личные профили. Используется кастомная модель пользователя с дополнительными полями (био, уведомления).
- **Главная страница (`main`):** Навигация, информация о проекте.
- **Админ-панель:** Полное управление контентом через стандартный интерфейс Django.
- **Static & Media:** Обработка статики через WhiteNoise и загрузка изображений товаров/пользователей.

## Технологический стек

- **Backend:** Python 3.8+, Django 4.2+
- **Database:** PostgreSQL
- **Frontend:** HTML5, SCSS (сборка в CSS), JavaScript (jQuery), FontAwesome
- **Инструменты:** django-debug-toolbar, whitenoise

## Структура проекта

- `FreakLabMarket/` — Основные настройки проекта (settings, urls).
- `main/` — Приложение для статических страниц (Home, About).
- `goods/` — Каталог товаров и категорий.
- `users/` — Управление пользователями, аутентификация и профили.
- `static/` — Статические файлы (стили, скрипты, шрифты).
- `media/` — Медиа-файлы (изображения товаров и аватары).
- `templates/` — HTML-шаблоны.
- `fixtures/` — Данные для первоначального наполнения базы.

## Установка и запуск

### 1. Подготовка окружения
Склонируйте репозиторий и создайте виртуальное окружение:
```bash
git clone <url-репозитория>
cd FreakLabMarket
python -m venv venv
# Windows:
venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate
```

### 2. Установка зависимостей
```bash
pip install -r requirements.txt
```
*(Если файла requirements.txt нет в корне приложения, убедитесь, что установлены django, pillow, psycopg2-binary, django-debug-toolbar, whitenoise)*

### 3. Настройка базы данных
Проект настроен на работу с **PostgreSQL**. Убедитесь, что у вас создана база данных (по умолчанию `freaklab`) и данные для подключения в `FreakLabMarket/settings.py` или `FreakLabMarket/secrets.py` верны.

### 4. Миграции и данные
Примените миграции и загрузите начальные данные:
```bash
python manage.py migrate
python manage.py loaddata fixtures/goods/categories.json
python manage.py loaddata fixtures/goods/products.json
```

### 5. Запуск
```bash
python manage.py runserver
```
Приложение будет доступно по адресу: [http://127.0.0.1:8000/](http://127.0.0.1:8000/)
